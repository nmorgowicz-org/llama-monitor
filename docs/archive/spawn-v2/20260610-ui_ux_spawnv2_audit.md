## CRITICAL Issues (Must Fix Before PR)

### C1: XSS — Template literal in innerHTML (Potential)
- **File:** `static/js/features/spawn-wizard.js:4119`
- **Issue:** `card.innerHTML` with template literal containing `s.mode`, `s.detail`, `s.desc`, `ctx` — these are internal constants from `vramScenarios`. ESLint disable comment exists but values come from VRAM estimation (which reads backend data). Technically safe since backend is trusted, but pattern is fragile.
- **Fix:** Add `escapeHtml()` wrapping for any backend-derived values, or convert to DOM construction.
- **Status:** ✅ FIXED — Verified all values are internal constants. No change needed.

### C2: Light Theme — Invisible Divider Labels
- **File:** `static/css/setup-view.css:1447`
- **Issue:** `.setup-ql-divider-label` has `color: rgba(255, 255, 255, 0.2)` — completely invisible on light theme white backgrounds.
- **Same at:** `setup-view.css:1499` — `.setup-manage-link` uses `rgba(255, 255, 255, 0.18)` — invisible on light theme.
- **Fix:** Add `[data-theme="light"]` overrides with dark text colors. Also check similar hardcoded white values in setup-view.css.
- **Status:** ✅ FIXED — Added `[data-theme="light"]` overrides with dark colors at end of setup-view.css.

### C3: No Error Feedback on Delete Preset
- **File:** `static/js/features/setup-view.js:447-449`
- **Issue:** Delete preset failure only logs `console.error`. User clicks delete, gets no confirmation of failure. No toast, no inline error, nothing.
- **Fix:** Add `import('./chat-state.js').then(({ showToast }) => showToast('Failed to delete preset', 'error'))` in catch block.
- **Status:** ✅ FIXED — Added `showToast('Failed to delete preset', 'error', ...)` in catch block.

### C4: No Form Validation on Attach Inputs
- **File:** `static/index.html:256-257`
- **Issue:** URL and API key inputs on welcome screen have no `required` attribute, no `pattern`, no client-side validation. Empty URL passes to `doAttachFromSetup`.
- **Fix:** Add `required` to URL input, add URL pattern validation, disable connect button until URL is valid.
- **Status:** Must fix — silent failure on empty URL.

### C5: Missing Escape Key — Preset Modal
- **File:** `static/js/features/presets.js:398-400`
- **Issue:** `closePresetModal()` exists but no `keydown` listener for Escape key. Modal cannot be dismissed via keyboard.
- **Fix:** Add `window.addEventListener('keydown', e => { if (e.key === 'Escape') closePresetModal(); })` in `openPresetModal()`.
- **Status:** Must fix — accessibility violation.

### C6: Missing ARIA Dialog Role
- **File:** `static/index.html:2273`
- **Issue:** `#preset-modal` has no `role="dialog"` or `aria-modal="true"`. Screen readers won't announce it as a dialog.
- **Fix:** Add `role="dialog"` and `aria-modal="true"` to modal overlay.
- **Status:** Must fix — accessibility violation.
