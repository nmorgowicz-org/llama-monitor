⚠️
Agent Started, HTTP Blocked
The agent started successfully but is not reachable from your browser. This typically means the remote machine's firewall is blocking HTTP access on port 7779.
Quick fixes:
  • Check Windows Defender Firewall (or equivalent) and allow port 7779
  • Verify the agent is listening on 0.0.0.0 (not just 127.0.0.1)
  • Try netsh advfirewall firewall add rule name="llama-agent" dir=in action=allow protocol=TCP localport=7779 on Windows
