    Speculative decoding i
    
                        
    
                            
    
                                 
                                 
                                
    
                                    Pick a mode to see a description.
                                    Shared hash pool n-gram: efficient for server use, good for general and roleplay. ~10–30% speedup, no extra files.
                                    Minimal-overhead n-gram: looks up last n-gram in history. Lightweight and safe for most models.
                                    Hash-map patterns: strong for repetitive or code-heavy content where longer sequences recur.
                                    Experimental: 4 values per key for complex repetition patterns. Best for very repetitive or structured outputs.
                                    MTP plus n-gram fallback: best for models with built-in MTP heads (Qwen3, DeepSeek, etc.). Requires --parallel 1. The MTP section above auto-selects this when enabled.
                                    Pure MTP: uses built-in prediction heads only. Requires --parallel 1. Use if you want MTP without n-gram fallback.
                                    Separate small draft model (1–3B) proposes tokens; main model verifies. Highest theoretical speedup (~2–3×) but requires an extra model file.

    Window size for n-gram lookup. Default: 24. Larger = longer patterns, more memory.
    
                                     
    
                                     
    
                                         
                                         
                                         
    Minimum draft tokens to propose per step. Default: 8.
    
                                     
    
                                     
    
                                         
                                         
                                         
    Maximum draft tokens per step; higher = more aggressive. Default: 24.
    
                                     
    
                                 
    
                             
    
                             
    
                                 On unified memory systems (Mac, DGX Spark, etc.), the KV prefix cache RAM (-cram) is set to 0 by default to avoid wasting shared memory.
